# docs_terms 使用说明

## 1. 仓库目的
- 为公司产品说明书提供统一的**中英文术语、标准句子和参数表**。
- 提高说明书编写效率和正确率。
- 适合小团队内部协作。

## 2. 目录结构说明
- `switch.md` / `router.md` / `mouse.md` ：各产品线的术语 + 句子 + 参数表。
- `README.md` ：使用说明文档。
- 所有内容都是 Markdown 格式，方便查阅和复制。

## 3. 使用流程

1. **编写说明书时参考仓库**
   - 打开对应产品线的 Markdown 文件
   - 查找术语词条和标准句子
   - 直接复制粘贴英文或中文到文档中
   - 避免创造未批准的新术语或句子

2. **新增或更新术语**
   - 在对应产品线 Markdown 文件末尾增加条目
   - 保持中英文对照、使用场景和参数说明完整
   - 提交 Git commit 记录变更

3. **维护参数表模板**
   - 参数表字段应保持标准英文名和中文名
   - 如果参数单位或默认值更新，统一修改 Markdown

4. **版本管理**
   - Git commit 用于追踪术语变更
   - 推荐每个季度或每次大版本更新打 Tag

5. **小团队协作建议**
   - 所有人统一使用这个仓库作为**唯一参考源**
   - 避免在本地文件随意创建新术语
   - 定期 review 和整理新增条目

## 4. 优点
- **统一标准**：中英文术语、句子、参数表固定
- **轻量易用**：一个 Markdown 文件即可查阅
- **可扩展**：未来新增产品线，只需增加新的 Markdown 文件
